# Microsoft Governance & Security Labs

A structured collection of hands-on labs and automated workflows focused on securing Microsoft infrastructure. This repository serves as a technical portfolio demonstrating how to bridge enterprise governance requirements with modern, automated cloud deployments.

## Repository Objectives
* **Policy as Code (PaC):** Deploying and managing Azure Policy definitions to enforce organizational guardrails.
* **Compliance Automation:** Integrating continuous compliance scanning directly into CI/CD pipelines.
* **Evidence Mapping:** Generating and exporting verifiable technical artifacts mapped to standard frameworks (ISO 27001, SOC 2).
* **Identity & Access:** Enforcing least-privilege models using Azure RBAC and Entra ID (formerly Azure AD).

## Lab Directory Structure

| Lab Module | Description |
| :--- | :--- |

| `02-iac-compliance-validation` | Pre-deployment infrastructure scanning to validate compliance against major frameworks. |
| `03-rbac-and-identity` | Architecting custom RBAC roles and configuring conditional access policies. |
| `04-evidence-automation` | Automating the collection of cloud state data to satisfy continuous audit requirements. |
